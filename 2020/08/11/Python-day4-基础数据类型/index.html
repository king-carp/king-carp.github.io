二、基本数据类型

整型 int

作用：记录个数，如人的年龄，出生年份，游戏等级等。

浮点数型 float

作用：记录带有小数的数字，如人的身高，体重，商品价格等。
定义方式：

整数型与浮点数型合成数字类型，在取出值后可进行数学运算。

字符串型 str

作用：记录描述性的状态，如名字，地，公司名称等。
定义方式：在两个单引号或两个双引号或两个三引号中包含一串字符。三引号可以定义多行字符。

如果要在单引号或者双引号中换行(如有时我们要把一个文件地址赋值给一个变量，因为文件地址太长)可以使用\来到下一行继续书写地址

注意：
所有引号的相同点：定义得到的都是字符串类型
所有引号的不同点：注意引号的嵌套，外层用双引号，则内层用单引号，三引号内的字符串可写多行。

补充：

列表类型 list

作用：用来记录多段的数据，如记录全班人的名字，存值时利用str型也可做到，但是在取出时，str型就会很麻烦,并且存入的元素可以保持自己原来的类型,存入字符串的话所有数据都会变成字符串型。
优点：可以按照索引取对应位置的值
定义方式：在[ ]内，以逗号为分隔符，元素之间没有任何关系，可以是任何类型，按从左到右依次存放多个值。并且列表一般含有多个值，在取名时利用复数形式可能更直观。

字典类型 dict

作用：与列表相同，它用于存放具有映射关系的数据，dict型的key具有描述性，有助于我们取出数据。字典类型同样可以保持数据的原类型
相同点：都能存放多个值，都能保持原数据的类型
不同点：列表用索引对应值，字典用key对应值，key通常为字符串类型。

定义：在{ }中，以逗号为分隔符，存放多个元素，每个元素以key:value的形式存放，value可以是任意类型，key通常为字符串类型。
优点：key通常对于value有描述性功能，方便记忆
